## RFC Process

### What is an RFC?

An RFC (Request for Comments) is a design document that describes a significant change or new feature for NeMo Curator. RFCs provide a consistent and controlled path for new features to enter the project.

### When to Write an RFC

Write an RFC for:
- New user-facing features
- Significant architectural changes
- Changes that affect multiple components
- Removal of features

### RFC Lifecycle

1. **Draft**: Initial proposal, open for feedback
2. **Review**: Active discussion and iteration
3. **Accepted**: Approved for implementation
4. **Implemented**: Feature has been merged
5. **Superseded**: Replaced by a newer RFC
6. **Rejected**: Not accepted for implementation

### RFC Template

```markdown
# RFC-NNN: Title

**Status:** Draft | Review | Accepted | Implemented | Superseded | Rejected
**Authors:** Name(s)
**Created:** YYYY-MM-DD
**Last Updated:** YYYY-MM-DD

## Summary
Brief description of the proposal.

## Motivation
Why is this change needed?

## Design
Technical details of the proposal.

## Implementation Plan
Phases and milestones.

## Open Questions
Unresolved issues.

## References
Related documents and resources.
```
